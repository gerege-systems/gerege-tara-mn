# Гэрэгэ — Нэгдсэн төлөвлөгөө (PLAN)

> Эстонийн **TARA**-г эх загвар болгон Монголын e-Засаглалд зориулсан нээлттэй эхийн
> нэвтрэлтийн үйлчилгээ бүтээх төлөвлөгөө.

## Нэг өгүүлбэрээр

> TARA-гийн "**нээлттэй OIDC цөм (Ory Hydra) + үндэсний адаптерын нимгэн давхарга**"
> загварыг хувилж, Монголын ХУР/ДАН/оператор дэд бүтцэд тааруулна.

## Шийдвэрлэгдсэн зүйлс

- **Нэвтрэлтийн аргууд:** eID карт, Mobile-ID, Smart-ID/App, Биометр (бүгд — зорилт)
- **MVP:** Hydra + Mobile-ID vertical slice (mock-оор эхэлнэ)
- **Стек:** Java 17 + Spring Boot + Ory Hydra (TARA-тай ижил)
- **Лиценз:** MIT (эх сурвалжийн уламжлалаар)
- **Нэр:** Гэрэгэ — эртний Монголын итгэмжлэлийн пайз

## Дараагийн тодруулга шаардах шийдвэрүүд

1. Сесс хадгалалт: **Redis** уу, **Apache Ignite** (TARA шиг) юу? → DevOps туршлагаас
2. ДАН-тай харьцах: **wrap** (зөвлөсөн) уу, **replace** уу? → бодлогын шийдвэр
3. Эхний бодит оператор: аль үүрэн оператортой эхэлж sandbox гэрээ хийх вэ?

## Дараагийн алхам (шууд эхлэх боломжтой)

1. `gerege-login/auth-api`-д `AuthenticationMethodHandler` интерфейсийг бэлдсэн —
   **LoA mapping логикийг та бичнэ** (доорх "Сурах горимын даалгавар" хэсгийг үз).
2. Docker Compose-оор Ory Hydra-г локалд босгох.
3. Mobile-ID handler-ийг `gerege-mock`-той холбож эхний end-to-end урсгал гаргах.

## Сурах горимын даалгавар (Learning mode)

`gerege-login/auth-api`-д **итгэмжлэлийн түвшин (LoA) тодорхойлох** логикийг үлдээсэн.
Энэ нь зүгээр нэг тохиргоо биш — **аюулгүй байдлын бодлогын гол шийдвэр**:

- Mobile-ID (SIM-PKI) = ихэвчлэн **HIGH** (хувийн түлхүүр SIM чипд)
- Биометр = маргаантай — нүүр таниулалт дангаараа **SUBSTANTIAL** уу, **HIGH** уу?
- eID карт = **HIGH** (физик чип + PIN)
