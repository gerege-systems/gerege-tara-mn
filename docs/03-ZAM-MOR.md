# 03 — Хэрэгжүүлэлтийн зам мөр (Roadmap)

> Стратеги: **Hydra + 1 арга (Mobile-ID) vertical slice** → дараа нь бусад адаптер.

---

## Үе шат 0 — Суурь (1-2 долоо хоног)

- [ ] Repo, лиценз (MIT), `NOTICE.md` (эх сурвалжийн зохиогчийн эрх)
- [ ] Maven multi-module скелет (`gerege-login`, `auth-api`, `auth-mobileid`, `gerege-mock`)
- [ ] Docker Compose: **Ory Hydra** + Postgres + Redis локалд ажиллуулах
- [ ] CI (GitHub Actions): build + test
- [ ] TARA-Login-ийн кодыг **уншиж судлах** (хувилахаас өмнө ойлгох)

**Үр дүн:** `docker compose up` → Hydra ажиллана, `/.well-known/openid-configuration` хариулна.

---

## Үе шат 1 — Vertical slice: Mobile-ID (3-4 долоо хоног) ⭐ MVP

- [ ] `AuthenticationMethodHandler` интерфейс тодорхойлох (`auth-api`)
- [ ] Hydra login/consent урсгалыг хэрэгжүүлэх (`core`) — TARA-гаас хувилж тааруулах
- [ ] `auth-mobileid`: эхлээд **`gerege-mock`-той** ажиллана (бодит оператор хараахан биш)
- [ ] Арга сонгох UI (Thymeleaf, MN/EN)
- [ ] End-to-end тест: жишиг RP → Гэрэгэ → mock Mobile-ID → id_token буцна

**Үр дүн:** Иргэн (mock) Mobile-ID-аар нэвтрээд жишиг үйлчилгээнд орно. **OIDC урсгал бүрэн батлагдсан.**

> 🎓 *Энэ бол хамгийн чухал үе шат — нэг арга бүрэн ажиллавал бусад нь зөвхөн "адаптер бичих" болж хувирна.*

---

## Үе шат 2 — Бодит интеграц (4-6 долоо хоног)

- [ ] **ХУР (X-Road)** клиент — иргэний РД-аар нэр, төлөв шалгах
- [ ] Бодит **оператор SIM-PKI** (Mobicom/Unitel/G-Mobile) холболт — гэрээ/sandbox шаардлагатай
- [ ] **OCSP + circuit breaker** (TARA-гаас хувилах)
- [ ] **Аудит лог** + мониторинг (Actuator, Prometheus)
- [ ] Аюулгүй байдлын review (security-review)

---

## Үе шат 3 — Бусад адаптер (зэрэгцээ, баг тус бүр)

- [ ] `auth-eid` — чипт иргэний үнэмлэх + Web eID төрлийн урсгал
- [ ] `auth-smartid` — банк/E-Mongolia апп (QR, push)
- [ ] `auth-biometric` — ХУР/ДАН нүүр таниулалт *(Монгол нэмэлт — TARA-д байхгүй)*

---

## Үе шат 4 — Production бэлэн байдал

- [ ] `gerege-admin` — RP бүртгэлийн портал
- [ ] Helm chart, K8s deploy, HA (high availability)
- [ ] Ачааллын тест, penetration test
- [ ] Баримт бичиг бүрэн (интеграцийн заавар RP-нарт)
- [ ] (Сонголт) **GovSSO** хэсэг нэмэх — нэгдсэн SSO + сесс хуваалцах

---

## Эрсдэл ба хараат байдал

| Эрсдэл | Бууруулах арга |
|--------|----------------|
| Оператор PKI-д хандах эрх/гэрээ | Эхлээд mock-оор бүтээж, гэрээг зэрэг ярих |
| ХУР-д холбогдох зөвшөөрөл | Төрийн байгууллагатай хамтран ажиллах |
| Хувийн мэдээллийн хууль зүйн нийцэл | Эрх зүйчтэй эрт зөвлөлдөх |
| Java/Spring туршлага хомс | TARA-гийн код + энэ нь сайн сурах материал |

---

## Амжилтын хэмжүүр (MVP)

1. ✅ Жишиг RP стандарт OIDC client-ээр Гэрэгэд холбогдоно
2. ✅ Иргэн нэг аргаар нэвтэрнэ, зөв `id_token` (РД, нэр, LoA) авна
3. ✅ Бүх нэвтрэлт аудит логд бүртгэгдэнэ
4. ✅ Гадны үйлчилгээ унахад circuit breaker ажиллана
